Unity笔记28-保存数据

Created2021-08-04|Updated2021-08-04|Unity学习笔记
|Post View:|Comments:

这一章比较简单，我们来学习如何保存数据，用到的有两个重要的API，文章中会有介绍到

认识API

这里我们用到两个API，一个是**PlayerPrefs类，虽然官方文档写的是EditorPrefs，但是他们的方法是一样的，另外一个要用到的API是JsonUtility.ToJson**方法

PlayerPrefs：用于本地持久化保存与读取的类，PlayerPrefs类工作原理非常简单，以键值对的形式将数据保存在文件中，然后程序可以根据这个名称取出上次保存的数值

JsonUtility.ToJson：生成对象的公共字段的 JSON 表示形式。

编写代码

我们新建一个SaveManager.cs然后来编写专门管理保存读取的类

using UnityEngine;

public class SaveManager : Singleton<SaveManager> {
    protected override void Awake(){
        base.Awake();
        DontDestroyOnLoad(this);//切换场景不销毁
    }

    void Update(){
        if(Input.GetKey(KeyCode.S))//保存
            SavePlayerData();

        if(Input.GetKey(KeyCode.L))//读取
            LoadPlayerData();
    }

    public void SavePlayerData(){
        var PlayerData = GameManager.Instance.playerStates.characterData;
        Save(PlayerData,GameManager.Instance.playerStates.characterData.name);
    }

    public void LoadPlayerData(){
        var PlayerData = GameManager.Instance.playerStates.characterData;
        Load(PlayerData,GameManager.Instance.playerStates.characterData.name);
    }

    void Save(UnityEngine.Object data,string key){
        var jsonData = JsonUtility.ToJson(data);//转换为json
        PlayerPrefs.SetString(key,jsonData);//再建立键值保存
        PlayerPrefs.Save();
    }

    void Load(UnityEngine.Object data,string key){
        if(PlayerPrefs.HasKey(key)){//如果键值存在的话就读取
            JsonUtility.FromJsonOverwrite(PlayerPrefs.GetString(key),data);
        }
    }
}

此时我们就可以进游戏体验一下了，按S键是保存玩家当前数据，按L键是读取，并且保存后在下一次游戏启动时仍然可以读取数据

Windows系统可以在注册表上看到保存的数据

跨场景保存读取

既然我们在下一次开启游戏也可以读取玩家数据，那么跨场景也不在话下

去到SceneController.cs修改一下代码

IEnumerator Transition(string sceneName,TransitionDestination.DestinationTag destinationTag){
        //保存数据
        SaveManager.Instance.SavePlayerData();
        
        if(SceneManager.GetActiveScene().name!=sceneName){
            //不同场景的话
            yield return SceneManager.LoadSceneAsync(sceneName);//协程加载场景
            yield return Instantiate(playerPrefab,
                GetDestination(destinationTag).transform.position,
                GetDestination(destinationTag).transform.rotation);//加载玩家素材对象
            
            //读取玩家数据
            SaveManager.Instance.LoadPlayerData();
            
            yield break;//加载好后跳出协程

            
        }else{
            //相同场景
            //先获得玩家的对象
            player = GameManager.Instance.playerStates.gameObject;
            //获得目的地传送门对象
            var td = GetDestination(destinationTag);
            //关闭导航移动
            player.GetComponent<NavMeshAgent>().enabled = false;
            //设置对象的坐标和旋转角度
            player.transform.SetPositionAndRotation(td.transform.position,td.transform.rotation);
            //恢复导航移动
            player.GetComponent<NavMeshAgent>().enabled = true;

            yield return null;
        }
    }

这里我们在加载另外一个场景前会先保存数据，在加载场景完后我们先读取数据再退出协程

这样就实现了人物数据的保存了，当然这只是保存读取了人物的生命值、经验值等数据，其它数据也可以自由发挥
