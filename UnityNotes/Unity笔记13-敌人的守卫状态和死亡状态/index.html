Unity笔记13-敌人的守卫状态和死亡状态

Created2021-07-19|Updated2021-07-19|Unity学习笔记
|Post View:|Comments:

这一章节我们来制作敌人的守卫状态和死亡状态

敌人的守卫状态（Guard）

打开EnemyController.cs

private Quaternion guardRotation;//记录敌人最初的旋转角度，用于脱离追击回去守卫点砖头转头

void Awake(){
    //...
    guardRotation = transform.rotation;//获得最初旋转角度
}

void SwitchEnemyStates(){
	//...

    switch(enemyStates){
        case EnemyStates.GUARD://守卫状态
            isChase = false;//脱离追击状态

            //如果在追击主角时回到守卫状态，就要跑回守卫点
            if(transform.position != guardPos){
                isWalk = true;//播放走路动画
                agent.isStopped = false;
                //导航回最初守卫点
                agent.destination = guardPos;

                if(Vector3.SqrMagnitude(guardPos - transform.position)<=agent.stoppingDistance){
                    //SqrMagnitude也是可以用于比较距离的方法，比起Distance稍微快一些
                    //这里是判断是否到达最初守卫点可以停止范围
                    isWalk = false;

                    //慢慢转头。第三个参数越接近1转得越快
                    transform.rotation = Quaternion.Lerp(transform.rotation,guardRotation,0.01f);

                }

            }
            break;
    }
}

这里我们就写好了守卫状态的代码，其中SqrMagnitude()作用和Distance()一样都是比较两个点的距离，然后我们还加了慢慢转头的效果，否则敌人回到守卫点是不会转头的

死亡状态

我们希望当敌人血量为0时播放死亡动画并且尸体消失掉

设置死亡动画

打开动画控制器，我们新建一个Death Layer，不要忘了权重设为1

这里有个GetHit是用来做暴击伤害受伤效果的，它有左边的Hit触发，而我们的死亡动画Die连着Any State表示任何状态都能转移过来，由布尔值Death控制

特别要注意的地方：关于死亡动画的设置Can Transition To一定不能勾选，不然会发生鬼畜

接下来主角的动画也是如此设置

编写动画状态转移代码

回到我们的EnemyController.cs中

private Collider collider;//碰撞体组件
private bool isDead;//死亡状态控制

void Awake(){
    //...
    collider = GetComponent<Collider>();
}

void Update(){
    if(charcterStates.currentHealth == 0)
        isDead = true;//血量变0则死亡状态为真
    //...
}

void SwitchAnimation(){
    //...
    anim.SetBool("Death",isDead);//是否播放死亡动画
}


void SwitchEnemyStates(){
    //第一个判断是否为死亡
    if(isDead){
        enemyStates = EnemyStates.DEAD;

    }else if(FoundPlayer()){//如果发现player 切换到CHASE
        enemyStates = EnemyStates.CHASE;
        //Debug.LogFormat("敌人{0}发现了玩家",this.name);//测试代码
    }
    
    
    case EnemyStates.DEAD:
    collider.enabled = false;//防止时候开业再次攻击
    agent.enabled = false;//直接关闭导航组件
    Destroy(this.gameObject,2f);//销毁游戏对象，延迟为2秒

    break;

}

这里我们设置了Death的动画状态，注意开头if(isDead)的下一个是else if，而Hit的状态可以去到CharcterStates.cs中设置

public void TakeDamage(CharcterStates attacker,CharcterStates defener){
    //计算伤害数值
    //当前攻击力减去目标的防御力，如果防御力过高就是0伤害
    int damage = Mathf.Max(attacker.CurrentDamage()-defener.currentDefence,0);
    currentHealth = Mathf.Max(currentHealth-damage,0);//血量也是最小值锁在0

    if(isCritical)//判断被打者是否受到暴击并且播放受伤动画
        defener.GetComponent<Animator>().SetTrigger("Hit");

    //TODO:打死怪后加经验
}

在中间我们打了一行代码，用来获取被打者的Animator并设置动画Hit状态，这样就可以集中管理暴击受伤动画了
