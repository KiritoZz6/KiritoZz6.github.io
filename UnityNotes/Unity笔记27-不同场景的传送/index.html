Unity笔记27-不同场景的传送

Created2021-08-03|Updated2021-08-03|Unity学习笔记
|Post View:|Comments:

这一章我们来实现不同场景的传送

首先，我们要把另外一个场景搭建好先，随后才可以进行下一步的工作，这里就自己发挥吧

搭建好场景后，我们就可以正式开始来实现如何跨场景传送了

设置好传送门属性

首先就是要把两个不同场景的传送门属性设置好，注意要选择不同场景传送并把名字填上

下面是主世界的传送门属性

以及另外一个世界传送门的属性

编写代码

下面的代码可能有点分散，需要修改的文件比较多，会有注释写出来在哪个文件

//SceneController.cs，添加了不同场景加载时的代码

public GameObject playerPrefab;//玩家素材对象，用来不同场景时进行加载

    public void TransitiToDestination(TransitionPoint transitionPoint){
        //考虑是同场景还是不同场景的传送
        switch(transitionPoint.transitionType){
            case TransitionPoint.TransitionType.SameScene:
                //同场景可以不用异步，直接传送
                //这里第一个参数是获得当前活动窗口的名字
                StartCoroutine(Transition(SceneManager.GetActiveScene().name,transitionPoint.destinationTag));
                break;
            case TransitionPoint.TransitionType.DifferentScene:
                //不同场景，用异步加载的方式
                StartCoroutine(Transition(transitionPoint.sceneName,transitionPoint.destinationTag));
                break;
        }
    }

IEnumerator Transition(string sceneName,TransitionDestination.DestinationTag destinationTag){
        //TODO:保存数据
        
        if(SceneManager.GetActiveScene().name!=sceneName){
            //不同场景的话
            yield return SceneManager.LoadSceneAsync(sceneName);//协程加载场景
            yield return Instantiate(playerPrefab,
                GetDestination(destinationTag).transform.position,
                GetDestination(destinationTag).transform.rotation);//加载玩家素材对象
            yield break;//加载好后跳出协程
            
        }else{
            //相同场景
            //先获得玩家的对象
            player = GameManager.Instance.playerStates.gameObject;
            //获得目的地传送门对象
            var td = GetDestination(destinationTag);
            //关闭导航移动
            player.GetComponent<NavMeshAgent>().enabled = false;
            //设置对象的坐标和旋转角度
            player.transform.SetPositionAndRotation(td.transform.position,td.transform.rotation);
            //恢复导航移动
            player.GetComponent<NavMeshAgent>().enabled = true;

            yield return null;
        }
    }

注意这里我们要外界把人物的素材拖进去代码中，以保证传送到另外一个场景时可以加载人物出来

修复问题

传送报错

此时我们传送时会报错，有好几个问题我们慢慢来看

第一个问题是要把场景放在Scene in Build的设置当中

第二个问题就是我们传送到新场景后，那些Manager文件都会不存在，因此我们要把它们加入DontDestory的清单里

//在GameManager.cs  SceneController.cs  MouseManager.cs加上这行代码
protected override void Awake()
{
    base.Awake();
    DontDestroyOnLoad(this);//防止此Manager文件在切换场景时给销毁
}

我们要在GameManager.cs SceneController.cs MouseManager.cs加上上面代码，并且是要继承单例模式的代码文件

人物传送后无法移动

这时候人物可以传送到另外一个场景了，但是不能移动，可能有下面问题

第一个可能存在的问题就是我们第二个场景的地板没有设置成Ground标签，我们要设置一下

第二个问题是我们在PlayerController.cs中的OnMouseClicked事件没有加载出来

void OnEnable(){

人物传送回来相机没有跟随

接着还有一个问题就是当我们从另外一个场景传送回来时，相机的跟随就会失效，这里来修复这个问题

//GameManager.cs
public void RigisterPlayer(CharacterStates player){
        //通过外界注册的方式来获得对象的playerStates
        this.playerStates = player;

        //外界人物对象注册同时让相机进行跟随
        followCamera = FindObjectOfType<CinemachineFreeLook>();//在地图中查找相机
        if(followCamera!=null){//如果找到相机就让它跟随
            followCamera.Follow = playerStates.transform;
            followCamera.LookAt = playerStates.transform;
        }

    }
