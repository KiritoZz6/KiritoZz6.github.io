Unity笔记22-制作血量条UI

Created2021-07-29|Updated2021-07-29|Unity学习笔记
|Post View:|Comments:

这一章我们来制作一下敌人的血量条

创建血条UI

创建血条基础形状（Canvas与Image）

首先在Hierachy窗口中创建一个Canvas（画布），并将它命名为HealthBarCanvas

之后我们改一下HealthBarCanvas设置，并把坐标重置

随后我们在这个画布的基础上创建一个Image，并将它命名为Bar Holder，然后在把它的坐标改一下，改成像一个血条的大小

我们就可以在Scene窗口上看到血条的初始样子了

创建血条底层图像（Source Image）

接着我们要为Bar Holder导入一个Source Image作为血条的底层，这里就选择自己创建一个图像作为它的底层

首先要打开Package Manager安装一下2D Sprite

接着我们创建一个路径专门存放UI相关的对象，这里创建一个Square方形图像

再把这个方形图像应用在刚刚的Source Image中就行了

由于现在我们创建出来的是一个血条的底层，那么我们可以把它的颜色设置为红色底，待会有个上层图像设置成绿色，这样弄出来就有血条的样子了

因此我们这里把颜色改为红色（这里我把颜色的透明度改成了 150 ）

创建血条上层图像

我们在Bar Holder的基础上再创建一个图像Image，命名为Current Health，并将宽度和高度设置成和底层一样（2.5和0.25）

同时Source Image还是和刚刚一样，颜色调为绿色

唯一不同的是图片的类型Image Type要调为填充Filled，填充的方式为水平填充Horizontal，之后我们就可以通过调节填充数量Fill Amount来控制绿色条的长度，从而代表血量的多少了

效果图如下

这样一个简陋的血条就弄好了，我们将它保存为一个素材

设置血条在敌人上方的位置

由于我们每个敌人高矮不同，就导致了血条在不同敌人的位置也不一样，同时也是为了方便我们后面的代码，这里我们要为每个敌人设置一下血条应该出现的位置

最直接的办法就是为每个敌人对象再创建一个子对象来记录血条位置

我们打开Prefabs ---> Characters目录编辑敌人素材，这里拿史莱姆为例子

像上面图片这样，我们创建了一个HealthBar Point的空对象来记录血条的位置，这样就很方便我们代码的编写了

最后按照同样的方式给每个敌人都安放HealthBar Point

编写血条代码

创建事件

在创建血条代码之前，我们先在CharacterStates.cs中创建一个事件，即在造成伤害时要触发的血条变动事件

//造成伤害时血条改变的事件
public event Action<int,int> UpdateHealthBarOnAttack;

//在造成伤害时调用
public void TakeDamage(CharacterStates attacker,CharacterStates defener){
    int damage = Mathf.Max(attacker.CurrentDamage()-defener.currentDefence,0);
    currentHealth = Mathf.Max(currentHealth-damage,0);
    Debug.LogFormat("{0}攻击了{1}，造成了{2}点伤害",attacker.gameObject.name,defener.gameObject.name,damage);
    if(attacker.isCritical)
        defener.GetComponent<Animator>().SetTrigger("Hit");
    
    //TODO:血量条UI
    UpdateHealthBarOnAttack?.Invoke(currentHealth,MaxHealth);//如果事件不为空则调用
}


public void TakeDamage(int damage,CharacterStates defener){
    int dam = Mathf.Max(damage - defener.currentDefence,0);
    defener.currentHealth = Mathf.Max(defener.currentHealth - dam,0);

    //血条事件
    UpdateHealthBarOnAttack?.Invoke(currentHealth,MaxHealth);//如果事件不为空则调用
}

创建血条代码

接下来我们就来写控制血条的脚本代码

在下图目录创建一个脚本，起名为HealthBarUI.cs

然后打开编写代码

public class HealthBarUI : MonoBehaviour{
    public GameObject healthUIPrefab;//供外界提供一个血条素材
    public Transform barPoint;//记录血量条该显示在敌人的哪个位置

    [Header("血条显示设置")]
    public bool alwaysVisible;//血条是否一直可见
    public float visibleTime;//血条显示时间（如果不是一直可见的情况下）
    private float timeLeft;//剩余的显示时间


    Image healthSlider;//拿到血条上层图像
    Transform UIbar;//记录生成的血条在的位置
    Transform cam;//拿到摄像机位置以处理血条一直面向我们摄像机
    CharacterStates currentStates;//获取当前人物的states

    void Awake(){
        currentStates = GetComponent<CharacterStates>();

        currentStates.UpdateHealthBarOnAttack += UpdateHealthBar;//注册事件
    }

    void OnEnable(){
        //OnEnable在每次切换地图时都会调用一次

        cam = Camera.main.transform;//获取主相机位置

        foreach(Canvas canvas in FindObjectsOfType<Canvas>()){
            //遍历每一个画布
            if(canvas.renderMode == RenderMode.WorldSpace){
                //找到血条所在的画布，有多种方法来获得
                //也可以供外界提供此画布，更严谨一点
                UIbar = Instantiate(healthUIPrefab,canvas.transform).transform;

                //获得上层图像（绿色条）的对象，下标为0
                healthSlider = UIbar.GetChild(0).GetComponent<Image>();

                //是否一直可见
                UIbar.gameObject.SetActive(alwaysVisible);
            }
        }
    }

    void UpdateHealthBar(int currentHealth,int maxHealth){
        //更新血量
        if(currentHealth <= 0){
            //如果当前血量小于等于0，就销毁UIbar
            Destroy(UIbar.gameObject);
        }

        //每次更新时必然可见
        UIbar.gameObject.SetActive(true);

        //得到血量百分比
        float sliderPercent = (float)currentHealth / maxHealth;
        //更新血条长度
        healthSlider.fillAmount = sliderPercent;

    }


    void LateUpdate(){
        //在Update后一帧执行
        //更新血条跟随敌人位置
        //这里先检测UIbar有没有给销毁
        if(UIbar != null){
            UIbar.position = barPoint.position;//更新到敌人位置
            UIbar.forward = cam.forward;//面朝摄像机（即摄像机的方向）

            //血条剩余显示时间
            if(timeLeft <= 0 && !alwaysVisible){
                timeLeft = visibleTime;
                UIbar.gameObject.SetActive(false);
            }
            else
                timeLeft -= Time.deltaTime;

        }
    }
}

最后把代码脚本挂载到每个敌人身上，再调整数据就可以运行试试看效果了
