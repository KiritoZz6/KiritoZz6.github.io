Unity笔记26-实现同场景的传送

Created2021-08-02|Updated2021-08-02|Unity学习笔记
|Post View:|Comments:

这里我们实现同场景的传送效果

调整传送门

在这之前，我们要把不同的传送门摆放好先

另外我们可能会发现点击不了传送门，这是因为传送门的box collider挡住了我们鼠标的射线，这里把它的大小改一下就行了

给传送门加上Portal的标签，然后再去MouseManager.cs修改一下鼠标的图样

void SetCursorTexture(){
		Ray ray = Camera.main.ScreenPointToRay(Input.mousePosition);
		if(Physics.Raycast(ray,out hitInfo)){
			//TODO:切换鼠标贴图
			switch(hitInfo.collider.gameObject.tag){
				case "Ground"://地面行走的图标
					Cursor.SetCursor(arrow,new Vector2(16,16),CursorMode.Auto);
					break;
				case "Enemy"://点击敌人的图标
					Cursor.SetCursor(attack,new Vector2(16,16),CursorMode.Auto);
					break;
				case "Portal"://点击传送门图标
					Cursor.SetCursor(doorway,new Vector2(16,16),CursorMode.Auto);
					break;
				default:
					Cursor.SetCursor(point,new Vector2(16,16),CursorMode.Auto);
					break;
			}
		}//if
			
	}

	void MouseControl(){
		if(Input.GetMouseButtonDown(0) && hitInfo.collider != null){
			if(hitInfo.collider.CompareTag("Ground")){
				OnMouseClicked?.Invoke(hitInfo.point);
				
			}
			if(hitInfo.collider.CompareTag("Portal")){
				//点击传送门
				OnMouseClicked?.Invoke(hitInfo.point);
			}
			if(hitInfo.collider.CompareTag("Enemy")){
				OnEnemyClicked?.Invoke(hitInfo.collider.gameObject);//点击敌人传递敌人的gameObject
			}
			if(hitInfo.collider.CompareTag("Attackable"))
				OnEnemyClicked?.Invoke(hitInfo.collider.gameObject);
		}
	}

编写传送管理代码

我们创建一个SceneController.cs，之所以不是用Manager是因为在Unity有个SceneManager同名的文件存在

注意此文件继承的是单例模式类

using System.Collections;
using UnityEngine;
using UnityEngine.SceneManagement;
using UnityEngine.AI;

public class SceneController : Singleton<SceneController>{

    GameObject player;//玩家
    public void TransitiToDestination(TransitionPoint transitionPoint){
        //考虑是同场景还是不同场景的传送
        switch(transitionPoint.transitionType){
            case TransitionPoint.TransitionType.SameScene:
            //同场景可以不用异步，直接传送
            //这里第一个参数是获得当前活动窗口的名字
            StartCoroutine(Transition(SceneManager.GetActiveScene().name,transitionPoint.destinationTag));
                break;
            case TransitionPoint.TransitionType.DifferentScene:
            //不同场景，用异步加载的方式
                break;
        }
    }

    IEnumerator Transition(string sceneName,TransitionDestination.DestinationTag destinationTag){
        //先获得玩家的对象
        player = GameManager.Instance.playerStates.gameObject;
        //获得目的地传送门对象
        var td = GetDestination(destinationTag);
        //关闭导航移动
        player.GetComponent<NavMeshAgent>().enabled = false;
        //设置对象的坐标和旋转角度
        player.transform.SetPositionAndRotation(td.transform.position,td.transform.rotation);
        //恢复导航移动
        player.GetComponent<NavMeshAgent>().enabled = true;

        yield return null;
    }

    private TransitionDestination GetDestination(TransitionDestination.DestinationTag destinationTag){
        //通过传送门目的地的标签寻找目的地传送门所在位置
        var entrances = FindObjectsOfType<TransitionDestination>();
        for(int i=0;i<entrances.Length;++i){
            if(entrances[i].destinationTag == destinationTag)//目的地传送门标签匹配的话
                return entrances[i];
        }
        return null;
    }

}

另外当我们玩家到达传送门，我希望它可以按键触发传送的，所以这里去到TransitionPoint.cs


//触发器设置
private bool canTrans;//是否触发传送

void Update(){
    if(Input.GetKeyDown(KeyCode.E) && canTrans){
        //按键触发传送
        SceneController.Instance.TransitiToDestination(this);
    }
}

还有一个工作就是要把我们的TransitionDestination.cs作为组件添加在传送门子对象的点上

这个点是用来设置当前的点是属于哪一个标签的，我们就把一个传送门当做入口设为ENTER，另一个传送门当做目的地设为A

另外去到传送门中，目的地也要进行对应的设置，入口传送门的目的地是A，反过来另外一个传送门想过来就要设置目的地为ENTER

注意：我们最好不要设置多个同一标签的传送门，可能会出现问题

最后，建立一个空对象，并把SceneController.cs作为组件附上去

Trigger无法触发问题

如果函数内的Trigger事件没有触发的话，可能是玩家上没有添加钢体的原因，把钢体添加在主动碰撞的物体上（这里是玩家），然后把传送门和玩家的碰撞体组件中IsTrigger勾选上就可以了
