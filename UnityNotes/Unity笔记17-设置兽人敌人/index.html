Unity笔记17-设置兽人敌人

Created2021-07-23|Updated2021-07-23|Unity学习笔记
|Post View:|Comments:

在这一章节中我们来设置一下兽人敌人，由于它的Attack02动画包含一个推人的效果，我们可以对它进行单独的测试，而不是继续沿用之前的EnemyController，但是我们可以在它基础上进行添加，所以可以选择继承类方式改写

继承EnemyController并添加推开人物方法

因为兽人是用继承的代码，我们要专门创建一个文件夹放特定敌人的EnemyController子类代码

同时也把这个代码挂载到兽人身上

public class Grunt : EnemyController{
    [Header("Skill（兽人特殊攻击设置）")]
    public float kickForce = 15;//推开敌人距离

    public void kickOff(){
        //如果访问不了attackTarget要返回父类修改为protect属性
        if(attackTarget != null){
            transform.LookAt(attackTarget.transform);

            Vector3 direction = attackTarget.transform.position-transform.position;
            direction.Normalize();//量化推开的方向

            attackTarget.GetComponent<NavMeshAgent>().isStopped = true;

            //velocity表示速度，这里给了主角一个反向速度，达到给推开的效果
            attackTarget.GetComponent<NavMeshAgent>().velocity = direction*kickForce;
            
            attackTarget.GetComponent<Animator>().SetTrigger("Dizzy");//人物进入眩晕状态

        }
    }

}

然后我们把动画事件添加上，Attack02用的事件是我们上面写的kickOff()

同时这里有个眩晕状态，我们把眩晕动画如下添加好，同时眩晕动画速度我改为了2，让它播放快一点

替换动画并且添加一个动画行为

我们把史莱姆的动画复制一份给兽人，然后为了保持兽人在推开人物时人物进入眩晕状态并且不能移动，我们还要添加一个动画行为（Behaviour）

替换动画过程不再描述，但是注意这里不能采用动画控制器继承的方式

这里就说一下动画行为，找到人物的Dizzy动画，点击Add Behaviour，创建一个叫做Stop Agent的行为，实际上就是一个代码文件，创建好后我们把这个代码文件归类放好

这个代码文件打开就已经有一些注释在那了，我们可以直接取消我们要用方法的注释，这里用到的就是进入动画、动画播放时以及动画结束时刻的代码

public class StopAgent : StateMachineBehaviour
{
    // OnStateEnter is called when a transition starts and the state machine starts to evaluate this state
    override public void OnStateEnter(Animator animator, AnimatorStateInfo stateInfo, int layerIndex)
    {
        animator.GetComponent<NavMeshAgent>().isStopped = true;
    }

    // OnStateUpdate is called on each Update frame between OnStateEnter and OnStateExit callbacks
    override public void OnStateUpdate(Animator animator, AnimatorStateInfo stateInfo, int layerIndex)
    {
        animator.GetComponent<NavMeshAgent>().isStopped = true;
    }

    // OnStateExit is called when a transition ends and the state machine finishes evaluating this state
    override public void OnStateExit(Animator animator, AnimatorStateInfo stateInfo, int layerIndex)
    {
       animator.GetComponent<NavMeshAgent>().isStopped = true;
    }

    // OnStateMove is called right after Animator.OnAnimatorMove()
    //override public void OnStateMove(Animator animator, AnimatorStateInfo stateInfo, int layerIndex)
    //{
    //    // Implement code that processes and affects root motion
    //}

    // OnStateIK is called right after Animator.OnAnimatorIK()
    //override public void OnStateIK(Animator animator, AnimatorStateInfo stateInfo, int layerIndex)
    //{
    //    // Implement code that sets up animation IK (inverse kinematics)
    //}
}

之后我们可以把行为放进人物的眩晕、受伤等动画，甚至是敌人的攻击动画

注意的是添加到敌人攻击动画时可能会有一个报错，原因是我们可能把敌人打死了，导致行为无法获得agent，解决办法如下，我们去到EnemyController.cs修改敌人死了之后的状态

case EnemyStates.DEAD:
collider.enabled = false;//防止时候开业再次攻击
//agent.enabled = false;//直接关闭导航组件
agent.radius = 0;//把agent的范围缩小
Destroy(this.gameObject,2f);//销毁游戏对象，延迟为2秒

这里我们原本想关闭agent以达到agent没有体积效果，所以可以把agent.enabled = false;语句改为了直接把agent范围缩小到0，也是同样的效果

到这我们就初步设置好了兽人了
