Unity笔记15-使用接口实现观察者模式的订阅和广播

Created2021-07-21|Updated2021-07-22|Unity学习笔记
|Post View:|Comments:

这一章我们使用接口来实现观察者模式的订阅和广播，同时也来实现玩家死了游戏结束之后敌人的获胜动画

编写接口

在Tools文件夹下我们新建一个代码文件IEndGameObserver.cs表示游戏结束之后的接口

public interface IEndGameObserver{
    //注意接口只写声明不写定义

    //游戏结束要操作的都在这个接口实现
    public void EndNotify();

}

然后我们来到EnemyController.cs编写一个广播方法

public class EnemyController : MonoBehaviour,IEndGameObserver{
    public void EndNotify(){
        //获胜动画
        //停止所有移动
        //停止Agent


        isChase = false;
        isWalk = false;
        attackTarget = null;

    }
}

其它要在游戏结束时要有操作的代码也可以像这种方式来继承接口并实现操作

订阅接口

因为我们会有很多不同的代码文件都会继承这个接口，怎样来管理订阅也是值得考虑的事情

打开上一章创建的GameManager.cs

public class GameManager : Sigltion<GameManager>{
    public CharacterStates playerStates;

    //游戏结束时调用的方法的数组
    List<IEndGameObserver>endGameObservers = new List<IEndGameObserver>();


    public void RigisterPlayer(CharacterStates player){
        //通过外界注册的方式来获得对象的playerStates
        this.playerStates = player;
    }

    public void AddObserve(IEndGameObserver gameObserver){
        //加入待操作数组
        endGameObservers.Add(gameObserver);
    }

    public void RemoveObserve(IEndGameObserver gameObserver){
        //将特定的观察者在待操作数组中删除
        endGameObservers.Remove(gameObserver);
    }

    public void NotifyObserves(){
        //调用待操作数组中每一个观察者
        foreach(IEndGameObserver gameObserver in endGameObservers){
            gameObserver.EndNotify();
        }
    }


}

因为我们是要在主角死了之后开始游戏结束的操作，所以回到PlayerController.cs在判断主角死了之后调用广播

void Update(){
    if(charcterStates.currentHealth == 0){//判断死亡
        isDead = true;
        GameManager.Instance.NotifyObserves();
    }
}

同时我们在EnemyController.cs中还要补充一个加入待操作数组方法和移除方法

//void OnEnable(){//Unity的一个启动游戏时自动调用的函数

void Start(){
    //开始游戏时先检测是否勾选了守卫状态
    if(isGuard){
        enemyStates = EnemyStates.GUARD;
    }else{
        enemyStates = EnemyStates.PATROL;
        GetNewWayPoint();
    }
	//FIXME:场景切换做好后修改掉
    GameManager.Instance.AddObserve(this);
}

void OnDisable(){
    if(!GameManager.IsInitialized) 
        return;//如果GameManager没有给构建的话就直接return
    
    GameManager.Instance.RemoveObserve(this);
}

由于OnEnable()是在游戏开始时早期就调用的，这时候我们这样写会发生报错，我们要等到后面写场景加载时才用这种方法，这里暂时把加入待操作数组代码单独写到Start()里

我们打开动画管理器，配置敌人在打死主角后的动画

新建一个Victory Layer（注意权重值为1），以及一个Win布尔值控制动画触发

返回EnemyController.cs代码我们要配置动画变量什么时候触发

void Update(){
    if(charcterStates.currentHealth == 0)
        isDead = true;//血量变0则死亡状态为真
    else{//加了个else是防止死后敌人在胜利动画下一帧又给切换回普通动画
        SwitchEnemyStates();
        SwitchAnimation();
        lastAttackTime-=Time.deltaTime;//每一帧都减冷却
    }
}
