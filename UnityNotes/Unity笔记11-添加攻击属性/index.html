Unity笔记11-添加攻击属性

Created2021-07-17|Updated2021-07-17|Unity学习笔记
|Post View:|Comments:

在这一章节中我们来给人物还有史莱姆添加攻击的属性，同时也给史莱姆补充上攻击的动作，那么我们就开始吧

创建有关攻击的ScriptableObject

首先还是新建文件夹和代码文件👇

打开这个代码，我们来写一下有关攻击的属性

[CreateAssetMenu(fileName = "New Data",menuName = "Attack/Attack Data")]
public class AttackData_SO : ScriptableObject{
    //记录攻击相关属性
    public float attackRange;//近战攻击范围
    public float skillRange;//远程攻击范围

    public float coolDown;//CD冷却

    public float minDamage;//攻击最小伤害
    public float maxDamage;//攻击最大伤害

    public float criticalMultiplier;//暴击加成百分比
    public float criticalChance;//暴击率

}

打完代码之后我们就可以建立一个有关攻击的ScriptableObject了

然后我们可以根据自己需求进行属性值设定

同时我们还要将它挂载到人物和敌人身上，我们打开Monobehavior文件夹下的CharcterStates.cs代码文件，这个代码文件是挂辅助将ScriptableObject挂载的

然后添加一行代码，接受外界提供的Player BaseAttackData的ScriptableObject

public AttackData_SO attackData;

之后我们就可以回到史莱姆上挂载了

补充之前的攻击相关代码

还记得之前在写人物和史莱姆攻击时留下来的一些待补充代码吗？我们现在就可以逐步来完善它们了

第一个要修改的是PlayerController.cs里面人物的攻击距离，之前我们一直是默认为1，但是现在我们可以来补充它了

//ScriptableObject相关变量
private CharcterStates charcterStates;

void Awake(){
    charcterStates = GetComponent<CharcterStates>();
}

IEnumerator MoveToAttackTarget(){
    agent.isStopped = false;
    transform.LookAt(attactTarget.transform);
    //TODO:添加人物攻击距离
    while(Vector3.Distance(transform.position,attactTarget.transform.position)>
          charcterStates.attackData.attackRange){
        //比较人物和敌人的距离是否大于0攻击距离，Distance()可以比较两个Vector3距离
        this.agent.destination = attactTarget.transform.position;

        yield return null;
    }
}

在声明一个CharcterStates后，我们要把这里的while判断条件右边的1改成charcterStates.attackData.attackRange以获取攻击距离判断

然后是EnemyController.cs，这里修改量比较大

//ScriptableObject相关变量
private CharcterStates charcterStates;//这个敌人攻击相关的

private float lastAttackTime;//上一次攻击的时间（用来实现CD冷却效果）

void Awake(){
    //...
    charcterStates = GetComponent<CharcterStates>();
    lastAttackTime = charcterStates.attackData.coolDown;//开始获取冷却时间
}

void Update(){
	//...
    lastAttackTime-=Time.deltaTime;//每一帧都减冷却
}


//判断攻击距离函数
bool TargetInAttackRange(){
    //人物在敌人的攻击距离内
    if(attackTarget!=null){
        return Vector3.Distance(attackTarget.transform.position,transform.position)
            <= charcterStates.attackData.attackRange;
    }

    return false;
}

bool TargetInSkillRange(){
    //人物在敌人的远程攻击距离内
    if(attackTarget!=null){
        return Vector3.Distance(attackTarget.transform.position,transform.position)
            <= charcterStates.attackData.skillRange;
    }

    return false;
}


void SwitchEnemyStates(){
    //...
    switch(enemyStates){
        case EnemyStates.CHASE://追击
            //...
            //TODO:在攻击范围攻击
            if(TargetInAttackRange()||TargetInSkillRange()){

                isFollow = false;//停止跟随
                agent.isStopped = true;//停下来

                //攻击冷却完成进行攻击，在前面声明了
                if(lastAttackTime < 0){
                    lastAttackTime = charcterStates.attackData.coolDown;

                    //暴击判断，随机获取的数值（0到1之间）在暴击率内则是暴击
                    //charcterStates.isCritical下面会讲，用来记录是否暴击的变量
                    charcterStates.isCritical = Random.value <
                        charcterStates.attackData.criticalChance;
                    //执行攻击
                    Attack();
                }
                
            }
            break;
    }
}

//攻击控制函数
void Attack(){

    transform.LookAt(attackTarget.transform);//攻击时敌人看向目标

    if(TargetInAttackRange()){
        //近身攻击动画

    }
    else if(TargetInSkillRange()){
        //远程或技能攻击动画
    }
}

我们看到中间控制攻击的时候有一个charcterStates.isCritical的新变量，这是在CharcterStates.cs中新声明的，可以供外界访问的布尔值，用来记录当前是否为暴击，同时虽然它是public属性，但是我们可以用[HideInInspector]将其隐藏起来

//CharcterStates.cs
public class CharcterStates : MonoBehaviour{
    [HideInInspector]
    public bool isCritical;//是否暴击
}

代码开始复杂了起来，需要好好理解哦

添加攻击动画

我们看到上面的代码有两个动画需要我们来添加，一个是近身攻击动画，一个是远程或技能动画

我们按上图这样编辑动画控制器，建立两个Trigger类型的变量来分别控制Attack01和Attack02的动画触发条件，其次要注意的地方就是由巡逻状态切换过去攻击状态时并不需要动画的过渡效果，同时记得不要忘记把触发条件选择上

然后我们回到代码中，找到控制攻击的Attack()函数补充一下

void Attack(){
    //控制敌人攻击的函数
    transform.LookAt(attackTarget.transform);//攻击时敌人看向目标

    if(TargetInAttackRange()){
        //近身攻击动画
        anim.SetTrigger("Attack");
    }
    else if(TargetInSkillRange()){
        //远程或技能攻击动画
        anim.SetTrigger("Skill");
    }
}

这样我们就实现了基本的攻击效果

我们还可以把暴击效果加上，可以把Skill动画换成我们的暴击效果动画

再次打开动画控制器，添加一个布尔类型的变量Critical

我们把Attack02当做暴击动画，然后调整一下动画的切换过程

这里注意一点就是还要区别普通攻击时Critical触发状态应该是false，以防动画冲突

然后返回代码，我们要时刻获取是否为暴击状态，那么可以如下操作

//实时更新Animator里面史莱姆的状态布尔值
void SwitchAnimation(){
    anim.SetBool("Critical",charcterStates.isCritical);//时刻获取是否为暴击
}

由于SwitchAnimation()是放在Update()里面运行的，所以可以把暴击状态放在这里让它每一刻都获取暴击状态

修复人物脱离敌人攻击范围后敌人不动问题

最后，我们有个小问题要修复一下

由于之前我们写了发现敌人时敌人就要停下来

agent.isStopped = true;

所以，如果人物脱离敌人攻击范围时我们就要把停止状态取消掉，不然就会出现敌人停留在原地的现象

case EnemyStates.CHASE://追击
//...

    if(!FoundPlayer()){
        //拉脱回上一个状态
        agent.destination = this.transform.position;//人物脱离追击范围后，敌人停在当前位置
        isFollow = false;
    }else{
        isFollow = true;//找到人物就跑着追
        agent.isStopped = false;//<-----------停止状态取消
        agent.speed=speed;
        //人物进到敌人的范围会给追击
        agent.destination=attackTarget.transform.position;
    }

那么到这，我们就完成了给人物和敌人添加攻击属性，并且还为敌人加了普通攻击和暴击等攻击动画，让游戏开始变得有模有样起来
