跳表项目总结

Created2022-07-01|Updated2022-07-02|面试
|Post View:|Comments:

跳表项目相对于WebServer比较简单，实际上就是实现了跳表这一个数据结构，其中每个节点存放的数据是一个Node<K,V>的结构，和哈希表有点类似

功能

该跳表主要功能有：

  • 插入元素
  • 删除元素
  • 显示列表
  • 将数据存放为文件类型
  • 加载文件数据到内存中

插入数据

跳表插入数据是先从第一列数组最上方开始，即左上角位置开始寻找插入位置，这个位置的层也是跳表的当前最高层

假设目前跳表层数为h，如果遇到当前层h没有合适位置插入，则需要往下一层h-1查找，逐渐查找到第0层的位置便是插入位置

而在每次从当前层往下层转换时，都会有个Node<K,V>* update[_max_level+1]节点数组来记录当前层的转换位置，例如在下列图示中，我们要插入50的值，而update数组则会记录：第四层的1、第三层的10、第二层和第一层的30

这是为了在最底层插入50后，随机判断上面的层是否要添加50这个数（提高查找效率），而这个随机判断法概率为1/2

/** 
*                            +------------+
*                            |  insert 50 |
*                            +------------+
* level 4     +-->1+                                                      100
*                  |
*                  |                      insert +----+
* level 3         1+-------->10+---------------> | 50 |          70       100
*                                                |    |
*                                                |    |
* level 2         1          10         30       | 50 |          70       100
*                                                |    |
*                                                |    |
* level 1         1    4     10         30       | 50 |          70       100
*                                                |    |
*                                                |    |
* level 0         1    4   9 10         30   40  | 50 |  60      70       100
*                                                +----+
*
*/

随机判断法在代码中函数名为int SkipList<K,V>::get_random_level()

代码如下：

//获取随机层数
template<typename K,typename V>
int SkipList<K,V>::get_random_level(){
    int k=1;
    while(rand()%2){
        ++k;
    }
    k=(k<_max_level)?k:_max_level;
    return k;
}

压力测试

插入数据

注意：顺序插入为for循环从0开始到数据总数逐个数插入（ for(0 ~insertNum) ）

随机插入则是在插入数据总数范围间取随机数（ rand()%insertNum ）

在设定跳表最高层为10层时：

顺序插入十万条数据平均耗时：210毫秒；随机插入十万条数据平均耗时：92毫秒

顺序插入五十万条数据平均耗时：3850毫秒；随机插入五十万条数据平均耗时：380毫秒

顺序插入百万条数据平均耗时：21473毫秒；随机插入百万条数据平均耗时：744毫秒

而在设定跳表最高层为20层时：

顺序插入十万条数据平均耗时：132毫秒；随机插入十万条数据平均耗时：92毫秒

顺序插入五十万条数据平均耗时：683毫秒；随机插入五十万条数据平均耗时：305毫秒

顺序插入百万条数据平均耗时：1443毫秒；随机插入百万条数据平均耗时：600毫秒
