暑假冲刺秋招项目--C++集群聊天服务器（预备知识篇）

Created2022-07-16|Updated2022-07-17|面试项目集群聊天服务器
|Post View:|Comments:

本C++集群聊天服务器项目于2022/7/16开始开发，为一个暑期项目

相比于之前的WebServer，本项目用到了JSON、muduo、nginx负载均衡、CMake等新功能

另外，本项目是在Windows环境中通过VSCode的SSH连接至虚拟机Linux开发的，也就是说本质上还是基于Linux环境

JSON

JSON不必多说，已经算挺熟悉的，它可以通过序列化和反序列化来传输或解析一个数据集，百度百科对于JSON解释如下

JSON（JavaScript Object Notation, JS对象简谱）是一种轻量级的数据交换格式。

它基于 ECMAScript（European Computer Manufacturers Association, 欧洲计算机协会制定的js规范）的一个子集，采用完全独立于编程语言的文本格式来存储和表示数据。

简洁和清晰的层次结构使得 JSON 成为理想的数据交换语言。 易于人阅读和编写，同时也易于机器解析和生成，并有效地提升网络传输效率。

回到来项目中，这里JSON用的是一个第三方库，只用引入一个json.hpp头文件就可以使用了，非常轻量方便

另外，这个json除了可以对普通数据类型数组、int、string等进行序列化外，还可以对STL中的vector<T>和map<K,V>进行序列化

普通数据类型：

json js;
// 添加数组
js["id"] = {1,2,3,4,5};
// 添加key-value
js["name"] = "zhang san";
// 添加对象
js["msg"]["zhang san"] = "hello world";
js["msg"]["liu shuo"] = "hello china";
// 上面等同于下面这句一次性添加数组对象
js["msg"] = {{"zhang san", "hello world"}, {"liu shuo", "hello china"}};

cout << js << endl;
//输出结果为：{"id":[1,2,3,4,5],msg":{"liu shuo":"hello china","zhang san":"hello world"},"name":"zhangsan"}

STL类型：

除了vector和map，该json库还能存储std::array, std::vector, std::deque, std::forward_list, std::list, std::set, std::multiset, std::unordered_set, std::unordered_multiset等容器

另外像std::map, std::multimap, std::unordered_map, std::unordered_multimap，nlohmann也是支持的，但是需要注意的是其中的Key被构造为std::string保存。

json js;
// 直接序列化一个vector容器
vector<int> vec;
vec.push_back(1);
vec.push_back(2);
vec.push_back(5);
js["list"] = vec;

// 直接序列化一个map容器
map<int, string> m;
m.insert({1, "黄山"});
m.insert({2, "华山"});
m.insert({3, "泰山"});
js["path"] = m;

cout<<js<<endl;
//输出结果为：{"list":[1,2,5],"path":[[1,"黄山"],[2,"华山"],[3,"泰山"]]}

另外，除了直接cout<<输出json序列化结果外，还可以将其转为string类型再输出

string jstr=js.dump();//dump()返回JSON对象中存储的原始string值。
cout<<jstr<<endl;

解析json（反序列化）功能操作如下：

// 模拟从网络接收到json字符串，通过json::parse函数把json字符串专程json对象
json js2 = json::parse(jsonstr);

这里只是展示了该JSON库的一小部分功能，更多的可以上网搜索：nlohmann json

muduo

muduo是陈硕大神个人开发的C++的TCP网络编程库。muduo基于Reactor模式实现。

使用muduo前还需要安装Boost库，这是由于muduo依赖于Boost库

muduo网络库给用户提供了两个主要的类

  • TcpServer：用于编写服务器程序的
  • TcpClient：用于编写客户端程序的

muduo有个好处就是它把网络I/O和业务代码区分开来了：

使用时我们只用关心用户连接和断开事件以及用户的可读可写事件就可以了，也就是说我们只用编写这两种事件的接口代码即可，其它如何连接、epoll、线程池等事项muduo帮我们完成了

使用muduo编写服务端时候代码模式比较固定，基本上只用修改用户连接事件回调函数、用户可读写事件回调函数以及服务端名称就行了

在编写完事件回调函数后，需要做的就是在服务端对象的构造函数中进行回调函数绑定，由于成员函数的参数有个隐式参数this的原因，固不能直接在调用setConnectionCallback()时将成员函数方法写进去里边

这里就要用到<functional>头文件中的std::bind()绑定了，同时也用到了std::placeholders::_1参数占位符

注意：由于回调函数是类的成员函数，成员函数有个隐式参数this，因此这里也要补上

_server.setConnectionCallback(std::bind(&ChatServer::onConnection,this,std::placeholders::_1));

下面是一个简单的使用muduo网络库服务端模板例子：

//基于muduo网络库开发的服务器程序实例
//muduo_server.cpp

#include <muduo/net/TcpServer.h>
#include <muduo/net/EventLoop.h>
#include <iostream>
#include <string>
#include <functional>

using namespace std;
using namespace muduo::net;
using namespace muduo;

/**
 * 1.组合TcpServer对象
 * 2.创建EventLoop事件循环对象指针
 * 3.明确TcpServer构造函数需要什么参数（有三个必要的），输出ChatServer的构造函数
 * 4.在当前服务器类的构造函数中，注册处理连接的回调函数和处理读写事件回调函数
 * 5.设置合适的服务端线程数量，muduo库会自己分配I/O线程和工作线程
 */
class ChatServer{
public:
    ChatServer(EventLoop* loop, //事件循环
                const InetAddress& listenAddr,  //IP+Port（端口）
                const string& nameArg)  //服务器名字
        :_server(loop,listenAddr,nameArg),_loop(loop)
    {
        //给服务器注册用户连接的创建和断开回调
        //因为回调函数onConnection是成员函数，有个隐式this指针
        //这就导致和原本setConnectionCallback要求的回调函数格式不一致
        //因此这里就要用到std::bind()绑定函数
        //_1为参数占位符
        _server.setConnectionCallback(
            std::bind(&ChatServer::onConnection,this,std::placeholders::_1));

        //给服务器注册用户读写事件回调
        _server.setMessageCallback(
            std::bind(&onMessage,this,_1,_2,_3));

        //设置服务端的线程数量（当>=2时会自动把其中一个作为I/O线程，其它为工作线程）
        //这里的4则表示 1个I/O 3个工作线程
        _server.setThreadNum(4);

    }

    //开启事件循环
    void start(){
        _server.start();
    }


private:
    muduo::net::TcpServer _server;  // #1.定义个服务器对象
    muduo::net::EventLoop* _loop;   // #2.事件循环指针

    //专门处理用户的连接和断开 回调函数
    void onConnection(const TcpConnectionPtr& conn){
        //打印连接或断开的客户端ip和port
        if(conn->connected()){
            //peerAddress对端信息
            cout<<conn->peerAddress().toIpPort()<<" -> "<<
                conn->localAddress().toIpPort()<<" state: online ."<<endl;
        }else{
            cout<<conn->peerAddress().toIpPort()<<" -> "<<
                conn->localAddress().toIpPort()<<" state: offline ."<<endl;
            
            conn->shutdown();//相当于close(fd)，注意是对端的fd
            //_loop->quit(); //关闭服务端
        }
    }

    //专门处理用户读写事件的回调函数
    void onMessage(const TcpConnectionPtr& conn,    //连接
                    Buffer* buffer,    //缓冲区
                    Timestamp time) //接收到数据的时间信息
    {
        string buf=buffer->retrieveAllAsString();//把接收的数据全部转为string
        cout<<"recv data: "<<buf<<"\ntime: "<<time.toString()<<endl;

        //原封不动返回数据给客户端
        conn->send(buf);

    }
};

int main(){
    EventLoop loop; //相当于创建了epoll
    InetAddress addr("127.0.0.1",6000);
    ChatServer server(&loop,addr,"ChatServer");//创建服务端对象

    server.start(); //相当于listenfd == epoll_ctl() ==> epoll
    loop.loop();    //相当于以阻塞调用epoll_wait，等待连接事件或读写事件
    
}

编译指令为：

# 注意 -lmuduo_net 要在 -lmuduo_base 前面
g++ muduo_server.cpp -o server -lmuduo_net -lmuduo_base -lpthread

CMake

CMake是一个跨平台的安装（编译）工具，可以用简单的语句来描述所有平台的安装(编译过程)

CMake使用简单方便，可以跨平台，构建项目编译环境。尤其比直接写Makefile简单（在构建大型工程编译
时，需要写大量的文件依赖关系），可以通过简单的CMake生成负责的Makefile文件。

这里以编译testMuduo项目为例来编写CMake实例，首先创建一个CMakeLists.txt文件，我们的CMake代码就在里面编写：

# CMakeLists.txt

cmake_minimum_required(VERSION 3.0)    #要求系统CMake最小的版本
project(main)   #工程名字

# 配置编译选项，这里是在CMAKE_CXX_FLAGS基础上后面再加个 -g
set(CMAKE_CXX_FLAGS ${CMAKE_CXX_FLAGS} -g)

# 配置头文件搜索路径
#　include_directories()

# 配置库文件搜索路径
# link_directories()

# 设置需要编译的源文件列表
set (SRC_LIST muduo_server.cpp)

# 把指定路径（这里.指当前目录）下的所有源文件名字加入到变量SRC_LIST中
# aux_source_directory(. SRC_LIST)

# 表示生成可执行文件server，由变量SRC_LIST源文件编译而来
add_executable(server ${SRC_LIST})


# 链接库，表示server程序由后面几个库链接（相比直接g++命令不用加-l）
target_link_libraries(server muduo_net muduo_base pthread)

写好后在当前目录下运行指令即可构建项目：

cmake .

上面这种方式会生成出来一大堆东西，其中就包括Makefile，之后我们通过make指令执行生成命令就好了

我们还可以改善一下，默认执行cmake .指令后，CMake生成了一大堆东西，我们可以将其分类放好，比如指定生成的程序放在哪个文件夹等等

在上面CMakeLists.txt添加一行代码即可：

# 设置可执行文件最终的存储目录在 当前目录/bin 下
set(EXECUTABLE_OUTPUT_PATH ${PROJECT_SOURCE_DIR}/bin)

同时别忘了要创建一个bin目录

弄好这两个事情后，make所生成的可执行文件就会在./bin目录下了

以小见大 — 一个项目合适的目录：

通常在github上看到别人的项目是有很多个文件夹分好类的，通常由以下几部分组成

  • bin：放可执行文件
  • lib：放程序生成的链接库
  • include：放项目的头文件
  • src：放项目的源代码文件
  • build：放编译的中间文件
  • example：放项目的实例文件
  • thirdparty：放第三方库，比如JSON库json.hpp
  • CMakeLists.txt
  • autobuild.sh：自己编写的自动生成项目的shell脚本
